# Full documentation set (north star)

The scaffold ships the high-leverage minimum. This is the exhaustive set a mature SaaS eventually maintains — add a doc when the project actually needs it, not before.

## Product
- Vision, MVP scope, roadmap *(scaffolded)*
- Feature PRDs *(scaffolded)*
- Personas / user research
- Pricing & packaging
- Competitive analysis

## Technical
- Architecture, data model, API conventions *(scaffolded)*
- ADRs, design docs *(scaffolded)*
- Sequence/flow diagrams
- Performance & scalability notes
- Integration guides (per third-party)

## Operations
- Runbook (deploy, rollback, common incidents)
- On-call / incident response
- Backup & restore procedure
- SLOs / monitoring & alerting catalogue
- Capacity & cost notes

## Security & compliance
- Baseline checklist *(scaffolded)*
- Threat model
- Data classification & retention policy
- Access control matrix
- Vendor / subprocessor list

## Quality
- Test strategy
- QA checklist / release criteria
- Accessibility statement

## Team / process
- Contributing guide
- Onboarding
- Coding standards
- Definition of done
- Release process

## Customer-facing
- User guide / help centre
- API reference (public)
- Changelog / release notes
- Status page policy
